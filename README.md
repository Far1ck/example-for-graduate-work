
# Приложение для управления объявлениями
Веб-приложение для размещения и управления объявлениями с системой аутентификации и комментариями.

## Описание проекта
Приложение предоставляет функционал для:

* Создания и управления объявлениями
* Регистрации и аутентификации пользователей
* Работы с комментариями к объявлениям
* Управления пользовательскими профилями
* Работы с изображениями

## Технологии и зависимости
* __Spring Boot__ 3.5.6
* __Java__ 17
* __Spring Security__ для аутентификации
* __PostgreSQL__ как база данных
* __Liquibase__ для миграций
* __Swagger__ для документации API
* __Lombok__ для сокращения шаблонного кода

## Установка и настройка
### Требования
* Java JDK 17
* Maven для сборки
* PostgreSQL сервер

### Настройка базы данных
Обновите параметры подключения в `application.properties`:

```
spring.datasource.url=jdbc:postgresql://localhost:5432/resaleDB
spring.datasource.username=student
spring.datasource.password=chocolatefrog
```

### Запуск
1. Клонируйте репозиторий
2. Выполните сборку:\
`mvn clean install`
3. Запустите приложение:\
`mvn spring-boot:run`

## Основные эндпоинты
### Управление объявлениями
* **GET /ads** - получение списка объявлений
* **POST /ads** - создание нового объявления
* **GET /ads/{id}** - получение объявления по ID
* **DELETE /ads/{id}** - удаление объявления
* **PATCH /ads/{id}** - обновление объявления
### Управление пользователями
* **POST /login** - аутентификация
* **POST /register** - регистрация
* **GET /users/me** - получение информации о текущем пользователе
### Управление комментариями
* **GET /ads/{id}/comments** - получение комментариев к объявлению
* **POST /ads/{id}/comments** - создание комментария
* **DELETE /ads/{adId}/comments/{commentId}** - удаление комментария

## Структура проекта
* **Controllers** - слой контроллеров
* **Services** - бизнес-логика
* **DTO** - объекты передачи данных
* **Entities** - модели данных
* **Config** - конфигурационные классы

## Безопасность
Приложение использует Spring Security с:

* Аутентификацией по JWT
* Ролевой моделью доступа
* Защитой от CSRF
* CORS настройками

## Документация API
Документация доступна через Swagger UI по адресу:

`/swagger-ui.html`
